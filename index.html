Welcome to FlexGP!

FlexGP is a Evolutionary Computation framework aimed at data modeling and knowledge discovery. Current release provides functionality both for modeling data and for testing the retrieved models. In this website we provide a quick tutorial to get started and model your data.

Tutorial

Step 1: Downloading the source code

$ cd your_repo_root/repo_name
$ git fetch origin
$ git checkout gh-pages

Step 2: Running FlexGP

It is possible to run FlexGP directly from your terminal or from Matlab

Running FlexGP from the terminal

Model the data

All you need to provide is the path to your dataset and the optimization time

$ java -jar flexgp.jar -train path_to_your_data -minutes 10

At the end of the run a set of files are generated:

  1. pareto.txt: models forming the Pareto Front (accuracy vs model complexity).

  2. leastComplex.txt: least complex model of the Pareto Front.

  3. mostAccurate.txt: most accurate model of the Pareto Front.

  4. knee.txt: model at the knee of the Pareto Front.

  5. bestModelGeneration.txt: most accurate model per generation.

  6. fusedModel.txt: fused model of the Pareto Front obtained with Adaptive Regression by Mixing (see Adaptive Regression by Mixing. Yuhong Yang. Journal of the American Statistical Association Vol. 96, No. 454 (Jun., 2001), pp. 574-588).

Test the models

Once the modeling is finished, it is possible to obtain the Mean Squared Error (MSE) of the models in files 1) to 5) with 

$ java -jar flexgp.jar -test path_to_your_data -integer true -scaled model_file

To obtain the MSE of the fused model of the Pareto Front:

$ java -jar flexgp.jar -test path_to_your_data -integer false -fused model_file

The -integer flag indicated whether the target values are integer or floating point values.

Running FlexGP from Matlab

Step 3: Speeding up your runs with C++ optimized execution

This option is only supported for Linux Debian platforms and requires the gcc and g++ compilers:

$ sudo apt-get install gcc
$ sudo apt-get install g++

Modify the Linux kernel parameter governing the shared memory size to be at least as large as the data being analyzed, in the next example we set it to 2GB

$ echo 2147483648 > /proc/sys/kernel/shmmax

$ sudo nano /sys/kernel/shmmax 
$ git fetch origin
$ git checkout gh-pages
