FlexGP: Flexible Symbolic Regression with Genetic Programing

FlexGP is a Genetic Programming framework aimed at data modeling and knowledge discovery. Current release provides functionality both for performing Symbolic Regression on numerical datasets and for testing the retrieved models. In this website we provide a quick tutorial on how to get started with FlexGP.

Tutorial

This release is only supported for Linux Debian platforms. The functionality of the current release is limited to Symbolic Regression tasks. The capability to perform binary classification will be added soon.

Step 2: Running FlexGP

In the current release, it is only possible to run FlexGP directly from your terminal (a Matlab wrapper will be included soon).

Running FlexGP from the terminal

Model the data

All you need to provide is the path to your dataset and the optimization time

$ java -jar flexgp.jar -train path_to_your_data -minutes 10

At the end of the run a set of files are generated:

  1. pareto.txt: models forming the Pareto Front (accuracy vs model complexity).

  2. leastComplex.txt: least complex model of the Pareto Front.

  3. mostAccurate.txt: most accurate model of the Pareto Front.

  4. knee.txt: model at the knee of the Pareto Front.

  5. bestModelGeneration.txt: most accurate model per generation.

  6. fusedModel.txt: fused model of the Pareto Front obtained with Adaptive Regression by Mixing (see Adaptive Regression by Mixing. Yuhong Yang. Journal of the American Statistical Association Vol. 96, No. 454 (Jun., 2001), pp. 574-588).

Test the models

FlexGP provides functionality to obtain the Mean Squared Error (MSE) of the retrieved models once the training is finished. In the examples below, the additional -integer flag indicates whether the targets are integer or floating point values. To test the models saved in files 1) to 5), do:

$ java -jar flexgp.jar -test path_to_your_data -integer true -scaled model_file

To obtain the MSE of the fused model of the Pareto Front:

$ java -jar flexgp.jar -test path_to_your_data -integer false -fused model_file

Step 3: Speeding up your runs with C++ optimized execution

This option requires the installation of the gcc and g++ compilers and the configuration of the Linux kernel parameter governing the maximum size of shared memory segments:

$ sudo apt-get install gcc
$ sudo apt-get install g++

Modify the Linux kernel parameter governing the maximum shared memory segment size to be at least as large as the data being analyzed, in the next example we set it to 2GB

$ sudo echo 2147483648 > /proc/sys/kernel/shmmax

To benefit from the optimized C++ execution, append the -cpp flag followed by the number of CPU threads that will be employed to speedup FlexGP (4 in the example below):

$ java -jar flexgp.jar -train path_to_your_data -minutes 10 -cpp 4
