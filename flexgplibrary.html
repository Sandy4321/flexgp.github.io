    FlexGP: the Java library

    FlexGP is a Java library that offers three services that simplify the deployment of EC algorithms in cloud environments:

    1. Auto-configuration of the core learner: exploration of input parameter space.

    2. Communication layer: message passing between cloud nodes to enable migrations.

    3. Runtime management and log generation: scheduling of learner and network tasks and log of received messages and intermediate results.

    

    Auto-configuration of the core learner

    FlexGP provides an auto-configuration service that allows to automatically explore the input parameters space of a given learner. Examples of these parameters are the crossover and mutation rate, or the population size among many others.

    The auto-configuration of each learner relies on two files containing the specification of the parameters in Factorize syntax. These two files need to be provided by the user since they are specific of each learner:

    1. The first files specifies the values of fixed arguments (such as dataset or number of threads) and the parameters that will be factored (these parameters will be explored in the cloud runs).

    2. In the second file, the possible values of the parameters are specified using the Factorize syntax.

    For further details of the Factorize Syntax and examples of use, please visit the this link.

    Communication layer

    FlexGP provides a TCP/IP communication layer that allows to pass messages between cloud nodes running the FlexGP library. This way, the FlexGP library enables migrations in cloud environments.

    Interactions with Cloud server

    The communication layer relies on two cloud server utilities provided by standard cloud management tools such as the euca2ools or the Amazon EC2 API tools:

    1. Self IP discovery: each cloud node running the FlexGP library self identifies itself at the beginning of the run.

    2. IP discovery of other FlexGP nodes: each node refreshes periodically the list of IP directions started with the same security keypair. This allows to detect when new nodes are added to a FlexGP run, thus providing elasticity to FlexGP.

    Message Passing

    Messages are passed via TCP/IP protocol. FlexGP abtracts the concept of message in such a way that it is possible to create, send, and receive different types of messages. In particular, the current FlexGP version employs two kinds of messages:

    1. Broadcast Messages: Every time a nodes refreshes its list of IPs, it pings all the retrieved IPs and filters unresponsive nodes, i.e. nodes that did not reply to the ping message. Such broadcast signals are used to maintain a list of responsive or “good” IP directions.

    2. Migrant Messages: this functionality allows FlexGP nodes to perform migrations. FlexGP provides an abstract Migrant Message class that needs to be extended and customized for each learner. This customization is necessary since different learners employ different representations.
