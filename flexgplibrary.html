    FlexGP: the Java library

    Overview

    FlexGP is a Java library that offers three services that simplify the deployment of EC algorithms in cloud environments. The philosophy of FlexGP is to facilitate the exploitation of the embarrassingly parallelism of Evolutionary Computation algorithms. In fact, the parallelism can be exploited at multiple levels:

    Run: several replicas of the algorithm can be executed in a concurrent manner with or without changes in the parameters of the algorithm.

    Data: ensemble approaches benefit from the the data-level parallelism by running concurrently EC instances that learn from different smaller subsets of the data.

    Populations: the population level parallelism can be exploited with the implementation of an island model in which disjoint sets of individuals evolve independently with the exception of infrequent migrations (exchanges of individuals)

    Evaluation: the evaluation of individuals of the same population can be performed simultaneously through the implementation of a Master-Worker model. Alternatively, the evaluation level parallelism can also be exploited by computing the predictions of a given model for independent test cases in parallel.

    While the implementation of the evaluation level parallelism depends on each specific learner, the first three levels of parallelism are generalizable for any EC algorithm. FlexGP offers the three following services:

  • Auto-configuration of the core learner: allow to explore different input parameters.

  • Communication layer: message passing between cloud nodes to enable migrations.

  • Runtime management and log generation: scheduling of learner and network tasks and log of received messages and intermediate results.

    • Auto-configuration of the core learner

    FlexGP provides an auto-configuration service that allows to automatically explore different input parameters for a given learner, thus exploiting the run level parallelism. Examples of these parameters are the crossover and mutation rate, the population size etc. The auto-configuration service also enables to learn from a subset of the data. Therefore, FlexGP relies on an ensemble approach in which each EC instance learns from a sub-sample of the exemplars and explanatory variables of the data. This approach can be graphically depicted as to sample both from the rows (exemplars) and columns (variables) of the training set. We have demonstrated that adopting this ensemble strategy provides robustness to our runs (see GECCO 2013, EuroGP 2014). The auto-configuration of each learner relies on two files containing the specification of the parameters in Factorize syntax. These two files need to be provided by the user since they are specific of each learner: 1. The first files specifies: - the values of fixed arguments such as dataset or number of threads. - the parameters that will be factored: these parameters will be explored in the cloud runs 2. In the second file, the possible values of the parameters are specified using the Factorize syntax. For further details of the Factorize Syntax and examples of use, please visit the this link.
