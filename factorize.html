    Enabling learner auto-configuration with the Factorize syntax

    The auto-configuration of each learner relies on two files containing the specification of the parameters in Factorize syntax. These two files need to be provided by the user since they are specific of each learner:

    1. The first files specifies the values of fixed arguments (such as dataset or number of threads) and the parameters that will be factored (these parameters will be explored in the cloud runs).

    2. In the second file, the possible values of the parameters are specified using the Factorize syntax.

    The Factorize syntax has been designed to specify the spectrum of possible values of the input parameters of any GP-based learner. The proposed syntax is aimed to be simple and intuitive. Note: all the brackets, words, numbers, and colons used in the following examples are separated with spaces.

    First file: specifying fixed and factored parameters

    The parameters of the learners can be whether fixed or factorized. The syntax employed to declare fixed parameters is: 
    learner parameter_name value
    while the following syntax is employed to set the parameters that will be factored: 
    factorizedParams{parameter_name_1,...,parameter_name_N}

    Example:

    learner threads 4
factorizedParams{selection,mutation_rate,crossover_rate,max_generation}

    Second file: Specifying possible values for the parameters

    Types

    FlexGP supports 4 basic types: 
    int, float, bool, string

    Default values

    For each parameter, a default value has to be specified. The following syntax has to be added after the type of the parameter: 
    default ( value )

    Discrete sets

    The instruction discreteSet is used to specify a set of discrete options for a given parameter. Note that the options can contain several words or tokens. The following notation must be used: 
    parameter_name type discreteSet default ( optiond ) { option1 ; option2 ; ... ; optionn }
    Note that all the brackets, words, and semicolons have to separated with spaces.

    Ranges

    This option only applies for the numerical types int and float. A range is characterized by a minimum value, a step value (difference between consecutive numbers of the set), and a maximum value. Factorize does not provide explicit support for continuous values. However, an easy approximation can be obtained by setting a small step. To specify a range of values for a given parameter, the following notation must be used: 
    parameter_name type range default ( val ) [ min : step : max ]

    Subsets of different sizes

    This instruction allows to select a subset of a varying size from a set of items. For instance, this instruction is helpful to select a subset of explanatory variables. The original set from which we will sample is specified with parameter_name while the second parameter specifies the size of the subset. To specify a random subset of varying size, the following notation must be used: 
    parameter_name type subset default ( 1 ) {val1 ; val2 ; ... ; valn }

    Comments

    Lines starting with the character ! will be ignored 
    ! this line will be ignored

    Examples

    
selection string discreteSet default ( rank ) { rank ; roulette } 
mutation_rate float range default ( 0.2 ) [ 0.0 : 0.05 : 0.2 ] 
crossover_rate float range default ( 0.7 ) [ 0.3 : 0.1 : 0.9 ] 
pop_size int discreteSet default ( 100 ) { 100 ; 500 ; 1000 } 
tournament_size int discreteSet default ( 2 ) { 2 ; 5 ; 10 } 
max_generation int range default ( 300 ) [ 50 : 50 : 500 ] 
fitness string discreteSet default ( cost ) { accuracy ; cost }
variables string subset default ( 1 ) { 0.25 ; 0.5 ; 0.75 ; 1 }
